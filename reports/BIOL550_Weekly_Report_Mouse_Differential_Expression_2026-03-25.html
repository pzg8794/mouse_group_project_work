Mouse RNA-Seq — Differential Expression (Family-Specific DESeq2)

BIOL550 — Weekly Report — Mar 25, 2026

What I accomplished since the previous report

This week the mouse RNA-seq project moved from alignment validation into differential expression analysis. Using the reverse-stranded count matrix from the canonical STAR run and the sample summary table from the local alignment workflow, I ran the DESeq2 workflow and reorganized the 26 samples into three family-specific models instead of one pooled comparison. This change was made in collaboration with the team so that sequencing-platform and sample-context differences would stay separated and the results would be easier to interpret.

I generated the DE package locally by adapting my previous DESeq2 scripts from my BIO630 course and organized the outputs into a shareable handoff for the team. For each family, I produced PCA plots, sample-distance heatmaps, dispersion plots, volcano plots, and ranked differential expression tables for the main contrasts. This gives the team a clear DE framework to use for interpretation and for the next phase of our project.

Methods used

DESeq2 example inside the R script
library(DESeq2)

dds <- DESeqDataSetFromMatrix(
  countData = counts_matrix,
  colData  = sample_table,
  design   = ~ condition
)

dds  <- DESeq(dds)
res  <- results(dds, contrast = c("condition", "injury", "naive"))
vsd  <- vst(dds, blind = FALSE)

Modeling strategy using DESeq2

  • Used reverse-stranded STAR gene counts as the count-layer input.
  • Used the alignment-stage sample summary as the DE metadata handoff.
  • Separated the dataset into three DE families:
    • family_tissue_novaseq6000
    • family_tissue_sham_novaseqx
    • family_neurons_novaseqx

This family-specific structure keeps platform and sample-context boundaries explicit instead of combining them in one confounded model. Within each family, only biologically interpretable contrasts were run, and all families were processed through the same DESeq2 workflow so the methods stayed consistent across the project.

Problems encountered

During this reporting period, the main challenge was deciding how to structure the DE analysis so the results would stay interpretable. A pooled model across all 26 samples would mix platform and sample-context effects, so the dataset had to be split into family-specific models. Another issue was that the sham and neuron contrasts produced weaker signals, so they are less useful than the tissue injury contrasts for the main report. Managing the data volume also remains ongoing, with SSRs, QC outputs, and DE tables being archived as they are generated to keep the analysis organized and in scope for the course.

For the contingency requirement, the workflow was also rebuilt with an alternate dataset so that, if the current analysis plan is not accepted, there is already a backup computational path. This was done to avoid losing time if the professor does not approve the current family-based DE structure. The contingency pipeline has already completed raw QC, fastp, and post-trimming MultiQC, with alignment currently running. Because GC-content behavior was one of the concerns in the current data, the GC-content comparison is included here as a quick check in the backup workflow. The main improvement so far is that adapter-related signal decreased after trimming, while duplication and base-content bias remain. These issues were handled through team coordination and shared review of the analysis plan. The main remaining dependency is professor feedback on the family-based DE strategy and on how the contingency pipeline should be used if the current approach is not accepted.
